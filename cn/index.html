atframework

Just start your game server cluster.

Start Using atframework   or   View on GitHub

高可用

自动连接和断线重连父节点和兄弟节点，自动按需连接，自动选择最佳连接方式。即便在同组服务内大量的节点使用共享内存通道，也能在不损失性能的情况下使用较少的内存消耗。

高性能

使用共享内存通道时有5千万QPS或1.5+GB的吞吐量。使用TCP网络通道时有2千五百万+的QPS或600+MB/s的吞吐量. 详情见libatbus的压力测试

去中心化

所有的服务都是去中心化的，所以可以很容易地平行扩展服务器集群和服务器组。

可定制化

MIT协议下，完全开源，可以自由定制服务。网关层也可以自由接入各种定制化协议。
